Running rucio-hermes

usage: rucio-hermes [-h] [--run-once] [--threads THREADS] [--bulk BULK]
                    [--delay DELAY] [--broker-timeout BROKER_TIMEOUT]
                    [--broker-retry BROKER_RETRY]

The Hermes daemon is responsible for delivering messages via STOMP to a messagebroker and via SMTP as email.

optional arguments:
  -h, --help            show this help message and exit
  --run-once            One iteration only
  --threads THREADS     Concurrency control: number of threads
  --bulk BULK           Bulk control: number of requests per cycle
  --delay DELAY         Delay control: second control per cycle
  --broker-timeout BROKER_TIMEOUT
                        Broker control: timeout second per cycle
  --broker-retry BROKER_RETRY
                        Broker control: number of retries per cycle

Create a message::

  $ python
  from rucio.core.message import add_message
  add_message(event_type='NEW_DID', payload='There is a new DID')

Run the daemon::

  $ rucio-hermes --run-once
